排序

  • 分析几种常见排序算法的流程、时间复杂度、空间复杂度、稳定性(即两个相同关键字的元素在初始列表中的相对先后次序在排序后是否能够保持)

1. 插入类排序

1.1 直接插入排序

将一个记录插入到已排好序的有序表中，从而得到一个新的、记录数增1的有序表。

如对数组A进行排序 A = {2, 1, 3}

初始化认为A[0]是一个有序序列，从A[1]开始进行插入排序得{1, 2, 3}，再循环插入排序过程得到最终的有序表。

代码实现：

/**
 * 直接插入排序.
 */
void StraightInsertionSort(int A[], int size) {
    for (int i = 1; i < size; ++i) {
        int insertNum = A[i];
        int j;
        for (j = i - 1; A[j] > insertNum && j >= 0; --j) {
            A[j + 1] = A[j];
        }
        A[j + 1] = insertNum;
    }
}

时间复杂度：最好O(n) (即初始列表有序情况)，最坏O(n^2)，平均O(n^2) 空间复杂度：O(1) 稳定性: 稳定

1.2 折半插入排序

由于插入排序的基本操作是在一个有序表中进行查找和插入，查找操作可以由折半查找来实现，由此进行的插入排序称之为折半插入排序(Binary Insertion Sort)

代码实现:

/**
 * 折半插入排序.
 */
void BinaryInsertionSort(int A[], int size) {
    for (int i = 1; i < size; ++i) {
        int insertNum = A[i];
        // 折半查找插入位置
        int left = 0, right = i - 1, mid;
        while (left <= right) {
            mid = (left + right) >> 1;
            if (A[mid] > insertNum) {
                right = mid - 1;
                continue;
            }
            left = mid + 1;
        }
        // 最终left>right, 插入位置即是left
        for (int j = i - 1; j >= left; --j) {
            A[j + 1] = A[j];
        }
        A[left] = insertNum;
    }
}

时间复杂度：最好O(n) (即初始列表有序情况)，最坏O(n^2)，平均O(n^2) 空间复杂度：O(1) 稳定性: 稳定

1.3 希尔排序(缩小增量排序 - Diminishing Increment Sort)

从对直接插入排序的分析得知，其算法时间复杂度为O(n^2)，但是若待排序记录序列已呈有序时，其时间复杂度可提高至O(n)。由此可设想，若待排序记录序列“基本有序”，直接插入排序的效率就可大大提高。从另一方面来看，由于直接插入排序的算法简单，在n较小时效率也很高。希尔排序就是从这两点出发进行改进得到的一种插入排序

将总待排序序列分为d个形如L[i, i+d, i+2d, … , i+kd]个子序列，对每个子序列进行直接插入排序，当整个序列中的元素“基本有序”时，对整个序列进行一次直接插入排序，一般地d1 = Floor(n/2)，di+1 = Floor(di/2) 直到dk = 1

代码实现:

/**
 * 希尔排序.
 */
void ShellSort(int A[], int size) {
    for (int d = size >> 1; d >= 1; d >>= 1) {
        for (int i = d; i < size; ++i) {
            int insertNum = A[i];
            int j;
            for (j = i - d; A[j] > insertNum && j >= 0; j -= d) {
                A[j + d] = A[j];
            }
            A[j + d] = insertNum;
        }
    }
}

时间复杂度：依据增量序列的选择会有所不同，具体的时间复杂度是一个待解决的数学问题，一般认为是O(n^1.3) 空间复杂度：O(1) 稳定性：不稳定

2. 交换类排序

2.1 冒泡排序

每一趟从后往前或从前往后两两比较相邻元素的关键字，如果是逆序，则交换二者的值，最终可以确定一个元素在序列的头部或尾部。比较n趟即可完成对整个序列的排序。

代码实现:

/**
 * 冒泡排序
 */
void BubbleSort(int A[], int size) {
    for (int i = 0; i < size - 1; ++i) {
        bool flag = false;
        for (int j = 1; j < size - i; ++j) {
            if (A[j - 1] > A[j]) {
                swap(A, j - 1, j);
                flag = true;
            }
        }
        // 如果一趟冒泡没有泡往上冒，说明该趟执行的序列已经是有序了，后面也没有必要继续执行
        if (!flag) {
            break;
        }
    }
}

2.2 快速排序

快速排序是对冒泡排序的一种改进。他的基本思想是：通过一趟排序将待排序记录分割成独立的两部分，其中一部分记录的关键字比另一部分的关键字小，则可分别对这两部分记录继续进行排序以达到整个序列有序。

代码实现：

/**
 * 快速排序-分区操作.
 *
 * @return 基准值的下标
 */
int Partition(int A[], int low, int high) {
    // 基准值, 将A[low]~A[high]中小于基准值的都放在其左边，大于基准值的都放在其右边
    int pivot = A[low];
    while (low < high) {
        while (low < high && A[high] >= pivot) {
            high--;
        }
        A[low] = A[high];
        while (low < high && A[low] <= pivot) {
            low++;
        }
        A[high] = A[low];
    }
    A[low] = pivot;

    return low;
}
/**
 * 快速排序
 */
void QuickSort(int A[], int low, int high) {
    if (low >= high) {
        return;
    }
    int pivot_idx = Partition(A, low, high);
    QuickSort(A, low, pivot_idx - 1);
    QuickSort(A, pivot_idx + 1, high);
}

时间复杂度: 最好：O(nlogn) 平均：O(nlogn) 最坏：O(n^2) 空间复杂度: O(logn) 稳定性：不稳定

3. 选择类排序

选择排序的基本思想是：每一趟在n-i+1(n = 1,2,…,n-1)个记录中选取关键字最小的记录作为有序序列中的第i个记录

3.1 简单选择排序

一趟简单选择排序的操作为：通过n-i次关键字间的比较，从n-i+1个记录中选出关键字最小的记录，并和第i个记录交换。

/**
 * 简单选择排序.
 */
void SelectSort(int A[], int size) {
    for (int i = 0; i < size; ++i) {
        int min_idx = i;
        for (int j = i; j < size; ++j) {
            if (A[j] < A[min_idx]) {
                min_idx = j;
            }
        }
        swap(A, i, min_idx);
    }
}

时间复杂度：O(n^2) 空间复杂度: O(1) 稳定性：不稳定 (如 2 2 1，经过一次选择排序后交换了第一个2和1的位置，两个2的相对次序改变)

3.2 堆排序

将待排序序列对应的一维数组看出是一棵完全二叉树，堆的含义表明完全二叉树中所有非终端节点的值均不大于(不小于)其左右孩子结点的值，由此堆顶元素必为序列中所有元素的最小值(最大值)。

3.2.1 堆的初始化

  1. 先将序列看成是一棵完全二叉树，然后从最后一个双亲结点开始向下调整(如果该双亲结点小于孩子结点，交换其与最大孩子结点的值，然后递归的对其孩子结点进行向下调整直到满足堆定义为止)，直至调整到根节点即代表堆初始化完成
  2. 进行排序，排序过程为交换大小为n的堆的堆顶元素和最后一个值，再对堆序列1~n-1的根节点进行向下调整，循环至堆的大小为1为止则排序完成
  3. 堆的插入，将插入结点放在末端，然后向上调整(如果其双亲小于插入的值，交换其次序，然后递归的对其双亲结点进行向上调整直到满足堆定义为止)
/**
 * 堆的向下调整.
 *
 * @param A 堆的一维数组存储
 * @param heapSize 堆的大小
 * @param adjustIdx 待调整的结点下标
 */
void AdjustDown(int A[], int heapSize, int adjustIdx) {
    // 按照完全二叉树的性质，最后一个双亲结点为树总结点数除以二向下取整, 此处由于数组下标从0开始, 故还要减一
    int lastParentIdx = (heapSize >> 1) - 1;
    // 如果要调整的结点非双亲结点，无需调整
    if (adjustIdx > lastParentIdx) {
        return;
    }
    // 求要调整的双亲结点的孩子结点的最大值，与该双亲结点比较并进行调整
    // 先初始化为其左孩子，因为由完全二叉树性质可以知道双亲结点一定有左孩子结点，未必有右孩子结点
    int max_child_val_idx = (adjustIdx << 1) + 1;
    if (max_child_val_idx + 1 < heapSize && A[max_child_val_idx + 1] > A[max_child_val_idx]) {
        max_child_val_idx++;
    }
    // 如果当前要调整的双亲结点的关键字已经比其孩子结点的关键字都要大，说明无需调整
    if (A[adjustIdx] > A[max_child_val_idx]) {
        return;
    }
    // 交换要调整的双亲结点的关键字和其最大关键字的孩子结点的值，使得这个结点满足大顶堆的定义
    swap(A, adjustIdx, max_child_val_idx);
    // 如果交换以后其孩子结点不符合大顶堆的定义，继续向下调整其孩子结点
    AdjustDown(A, heapSize, max_child_val_idx);
}

/**
 * 堆排序.
 */
void HeapSort(int A[], int size) {
    // 初始化堆
    int i = (size >> 1) - 1;
    for (; i >= 0; --i) {
        AdjustDown(A, size, i);
    }
    // 开始排序
    for (i = size - 1; i >= 0; --i) {
        swap(A, i, 0);
        AdjustDown(A, i, 0);
    }
}

时间复杂度：O(nlogn) 空间复杂度：O(1) 稳定性：不稳定

4. 归并排序

归并的含义是将两个或两个以上的有序表组合成一个新的有序表

  1. 将列表分为n个子表，每个子表只有一个元素，故每个子表必须有序
  2. 将n个子表归并为n/2个子表
  3. 循环2直至整个列表有序
/**
 * 归并. 
 */
void Merge(int A[], int low, int mid, int high) {
    int size = high-low+1;
    int *B = (int *)malloc(sizeof(int) * size);
    int i = 0;
    int ls = low;
    int rs = mid + 1;
    while (ls <= mid && rs <= high) {
      if (A[ls] <= A[rs]) {
        B[i++] = A[ls++];
      } else {
        B[i++] = A[rs++];
      }
    }
    while (ls <= mid) {
      B[i++] = A[ls++];
    }
    while (rs <= high) {
      B[i++] = A[rs++];
    }
    for (i = low; i <= high; ++i) {
      A[i] = B[i-low];
    }
    free(B);
}

/**
 * 归并排序. 
 */
void MergeSort(int A[], int low, int high) {
  if (low >= high) {
    return;
  }

  int mid = (low + high) >> 1;

  MergeSort(A, low, mid);
  MergeSort(A, mid + 1, high);
  Merge(A, low, mid, high);
}

时间复杂度：O(nlogn) 空间复杂度: O(n) 稳定性：稳定

5. 基数排序

原理是将整数按位数切割成不同的数字，然后按每个位数分别比较。基数排序的方式可以采用LSD（Least significant digital）或MSD（Most significant digital），LSD的排序方式由键值的最右边开始，而MSD则相反，由键值的最左边开始。

将待排序整数的总数记为n，所有整数位数的最大数量记为d，每位值的取值范围大小记为r

思想：

  1. 初始化r个队列，队列按次序排序，比如要对十进制数排序可以初始化十个队列，分别是Q1~Qr
  2. 遍历所有待排序整数，按整数的最右边第一位依次入到不同的队列，比如整数123，因为其最右边的位为3，故该整数入队Q3
  3. 按Q1~Qr的次序出队所有元素得到一个序列
  4. 遍历3)得到的序列，按每个整数的最右边第二位一次入队
  5. 循环直到所有位都入队、出队过，得到的最终序列即是一个有序序列

时间复杂度：O(d*(n+r)) 空间复杂度：O(r) 稳定性：稳定
