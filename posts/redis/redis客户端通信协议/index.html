Redis客户端通信协议

  • Redis客户端与服务端之间的通信协议是在TCP协议之上构建的
  • Redis制定了RESP（Redis Serialization Protocol - Redis序列化协议) 实现客户端与服务端的正常交互

1. 发送命令格式

RESP规定一条命令的格式如下： *<参数数量> CRLF $<参数1的字节数量> CRLF <参数1> CRLF … <参数N> CRLF 以 set hello world命令为例：

*3  # 有三个参数
$3 # 第一个参数 set 的字节数为3
set # 第一个参数
$5 # 第二个参数 hello 的字节数为5
hello # 第二个参数
$5 # 第三个参数 world 的字节数为5
world # 第三个参数

2. 返回结果格式

Redis的返回结果类型分为以下五种

  • 状态回复：在RESP中第一个字节为 +
set hello world
+OK
  • 错误回复：在RESP中第一个字节为 -
smembers hello
-WRONGTYPE Operation against a key holding the wrong kind of value
  • 整数回复：在RESP中第一个字节为 :
set hello 1
+OK
incr hello
:2
  • 字符串回复：在RESP中第一个字节为 $
get hello
$1
2
  • 多条字符串回复：在RESP中第一个字节为 *, (如mget等)
mget hello hello1
*2
$1
2
$1
3
